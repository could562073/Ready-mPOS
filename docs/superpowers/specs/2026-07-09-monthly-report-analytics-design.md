# 月結分析對帳報表設計（月結頁原地強化）

> **Status**: 設計已核准（2026-07-09，方案 A + 公休規則定案）
> **Scope**: 只動 `MonthlyReportPage` 與新增純函式/儲存模組；首頁（Dashboard）本期不動
> **分支**: 沿用 `feature/line-item-transactions-redesign`（cutover 前工作都疊在此分支，避免分支疊分支；cutover 後新功能才依 Git 分支流程開獨立分支）
> **版本**: 完成後 bump `2.0.0-beta.3`

## 背景與需求（來自真實餐廳老闆）

使用者原以紙本記錄日損益與月損益，每月底手動核對。逐筆交易改造（Phase 1–7）後
「帳目」頁已滿足日常記帳，但月結頁分析偏薄。需求訪談結論：

1. **防漏記**：月底核對很大一部分在抓漏記——要自動標出「完全沒記帳的日子」，
   且固定公休日不能被誤報。
2. **成本結構**：二級分類目前完全沒被分析利用——要能展開一級看二級細目；
   要看每類支出**佔收入比**（食材率/人事率）；要能**與上月比較**抓異常變化。
3. **賺多少**：本月淨額要能與上月比較。
4. **明確不做**：異常日偵測、逐日打勾覆核、報表匯出（現有月結頁右上的匯出
   stub 按鈕**移除**）、首頁改動（使用者認為首頁目前存在意義存疑，另案檢討）。

## 設計總覽（方案 A：現有月結頁原地強化）

保持現有骨架（月份選擇器 → Hero → 圖表/明細切換 → 趨勢圖 + 分類卡），插入與升級：

```
月份選擇器（移除右上匯出 stub）
Hero 本月淨額
  └ 新增一行：vs 上月 +$3,200（+8%）
【新】未記帳日提示卡（無漏記時整卡不渲染）
圖表/明細切換（不動）
趨勢圖（不動）
成本結構卡（升級自「本月分類」卡）
  ├ 每列新增：vs 上月增減
  ├ 支出列新增：佔收入比
  └ 點列展開：二級細目
每日明細表（不動）
```

落選方案：三分頁改版（總覽/成本分析/明細）——資訊空間較大但導覽多一層、
改版範圍大，目前功能量撐不起，YAGNI。

## 1. 公休日規則（兩層）

**固定週公休**（設定頁新增「每週公休日」，可複選週日～週六、可不設）＋
**臨時逐日標**（月結漏記卡上對單日點「標為公休」）。固定公休日永久不算漏記；
臨時休（連假、臨時事件）逐日標。

儲存（`lib/closedDays.ts`，localStorage、本機不跨裝置同步——月結核對通常在
一台主力機上做，先不動 `_config`；日後有跨裝置需求再加）：

- `mpos_weekly_closed`：`number[]`（0=週日 … 6=週六）
- `mpos_closed_days`：`string[]`（`'YYYY-MM-DD'`，臨時公休）

API：`getWeeklyClosed / setWeeklyClosed / getClosedDays / markClosed / unmarkClosed`。

## 2. 資料計算層（新增 `lib/monthReport.ts`，全純函式 + Vitest）

- **`missingDays(month, txDates, weeklyClosed, closedDays, today)`**
  回傳該月「完全沒交易」的日期清單。排除：固定週公休、已標臨時公休、
  未來日（當月只算到 `today`；歷史月算整月）。
- **`sumByCategoryAndSub(txs, categories)`**
  一級類別 → `{ 總額, 二級細目 }` 彙總（直接從交易算——現有
  `buildDailyRecordsFromTx` 合成時丟掉 `subId`，不能重用）。無二級的交易
  歸入「（未分類）」細目；二級名從 `Category.subs` 解析，dangling `subId`
  一樣落入「（未分類）」。
- **`comparisonRange(month, today)`**
  決定「與上月比較」的基準：查看**進行中的月份** → 上月同期（上月 1 號～同日，
  上月天數不足則取上月最後一天）；查看**已結束月份** → 上月全月。
  跨年（1 月 vs 去年 12 月）在此處理。
- **delta 計算**：本月值 − 上月基準值，含金額差與百分比（上月為 0 時只顯示
  金額差不顯示 %，避免除以零/無意義的 ∞%）。

## 3. UI（只動 `MonthlyReportPage` + `SettingsPage` 公休設定）

- **Hero**：淨額大字下加一行小字 `vs 上月同期 +$3,200（+8%）`（歷史月顯示
  `vs 上月`）。收入為綠/支出紅的語意不適用於淨額——增=綠、減=紅。
  上月完全無資料則整行不顯示。
- **未記帳日卡**（Hero 下方）：無漏記 → 不渲染。有漏記 → 琥珀色提示卡，
  標題「7 月未記帳：N 天」，日期 chips（`7/18（五）`）。點 chip 出兩動作：
  「去補記」（`onSelectDate(date)` 跳帳目頁該日）／「標為公休」（立即從卡中
  消失）。卡底小字「本月已標公休 N 天」可展開列出並取消誤標。
- **成本結構卡**（升級現有 `CategoryBars`）：保留收入/支出分組與佔比橫條，
  每列新增 vs 上月增減（支出增=紅、支出減=綠；收入相反；持平灰「─」；
  上月無此類別顯示「新」）；支出列額外顯示佔收入比（`食材採購 34%`，
  分母=本月總收入毛額；總收入為 0 不顯示）。點列展開二級細目縮排小列
  （二級名、金額、佔該一級 %），再點收合。無二級資料的類別不可展開。
- **設定頁**：「每週公休日」區塊（日～六 chips 多選）。
- **移除**：月結頁右上匯出 stub 按鈕。

## 4. 資料流

頁面在現有 `useMonthTransactions(month)` 之外多查一次
`useMonthTransactions(prevMonth)`（Dexie 索引查詢，成本可忽略）；本月/上月
交易分別餵純函式得彙總與 delta。公休設定讀 localStorage 進 state，
標記/取消即時重算未記帳卡。**不動同步層、不動 DB schema、不動 Dexie 版本。**

## 5. 測試

- **Vitest**：`monthReport.ts`（進行中月份只算到今天、固定週公休排除、
  臨時公休排除、跨年比較、上月天數不足、二級歸戶含未分類與 dangling subId、
  上月為 0 的 delta）＋ `closedDays.ts` CRUD。
- **Playwright E2E**：帳目新增一筆帶二級的支出 → 月結成本結構展開見二級細目；
  未記帳日卡出現 → 標公休 → 該日從卡中消失且 reload 後持久。
- 既有 tsc / Vitest / build 全綠為合併門檻。
