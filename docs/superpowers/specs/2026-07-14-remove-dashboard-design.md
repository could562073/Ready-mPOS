# 移除首頁（Dashboard），獨有指標搬進帳目頁 — 設計文件

> **日期**: 2026-07-14
> **狀態**: 已核准（brainstorming 方案 A）
> **目標版本**: 2.2.0（MINOR）
> **分支**: `feature/remove-dashboard`（worktree: `worktree-remove-dashboard`）

## 背景與動機

使用者確認：打開 App 的目的就是記帳／看帳目，「首頁」只是路過頁。

自 Phase 6 把落地頁改為「帳目」（月曆＋逐筆列表）、2.1.0 強化「月結」（Hero vs 上月、
成本結構卡、未記帳日卡）後，首頁 8 塊內容約 6 成已被兩頁覆蓋：

| 首頁內容 | 已被誰覆蓋 |
|---|---|
| 今日收入來源分解／今日支出明細 | 帳目頁單日逐筆列表（更細） |
| 本月淨額小卡、近 7 天收入圖＋漲跌 badge | 月結頁 Hero vs 上月、TrendChart |
| 「編輯今日帳目」快捷 | 落地頁即帳目頁＋FAB |
| 問候語、同步狀態 pill | 底部導覽列已有「同步中…」提示 |

首頁**獨有**、值得保留的：

1. **今日淨額（扣分潤後）** — 帳目頁小計與月曆格皆為毛額，不扣手續費
2. **外送佔比 >40% 洞察卡** — 提醒老闆推內用提升毛利

決策：移除「首頁」tab，上述兩個獨有指標搬進帳目頁單日小計區。

## 設計

### 1. 導覽與 App.tsx

- `NAV_ITEMS` 移除 `dashboard` 項 → 導覽剩 **帳目／月結／設定** 三個 tab（順序不變）。
- `Tab` 型別移除 `'dashboard'`（`App.tsx` 內宣告；`DashboardPage` 自帶的本地宣告隨檔案刪除）。
- 移除 `DashboardPage` import、`tab === 'dashboard'` 渲染分支、`handleNavigate`
  （其唯一用途為首頁「編輯今日帳目」按鈕的導頁 callback）。
- **刪除 `frontend/src/pages/DashboardPage.tsx`**。不比照 `DailyEntryPage` 暫留：
  當時尚在過渡期，現 cutover 已完成、無任何引用，git 歷史可隨時找回。

### 2. 帳目頁單日小計區強化（指標搬家）

現行小計為兩張卡（收入合計／支出合計，僅在該日有交易時顯示），改為：

- **第三張卡「淨額（扣分潤）」**：`收入合計 − 支出合計 − 平台費`。
  - 平台費 = 該日**收入**交易金額 × 所屬一級類別的 `fee`（`fee > 0` 的類別才計）。
  - 全部類別皆無 `fee > 0` 時，label 顯示「淨額」（與首頁原行為一致）。
  - 三張卡同列，字級較現行兩卡略縮以塞入手機寬度（maxWidth 430）。
  - 顯示條件沿用現行（該日有交易才顯示）。
- **外送佔比洞察卡**：該日 `fee > 0` 類別收入合計佔當日總收入 >40% 時，
  於小計下方顯示提示卡（沿用首頁文案：類別名稱、佔比 %、已扣手續費金額、建議推內用）。
- 計算邏輯抽**純函式**進 `lib/aggregate.ts`（Vitest 覆蓋）：
  - `dayFeesFromTx(txs, categories)` → 當日平台費合計
  - `dayFeeRatio(txs, categories)` → fee 類別收入佔比（總收入為 0 時回傳 0）
  - 首頁原本把手續費邏輯內聯在元件裡，這次順手歸位到純函式層。
- 指標適用**任一選定日**（不只今天）：點過去某天也能看到該日扣分潤後淨額，
  比原首頁只能看今天更有用。

### 3. 明確捨棄（不搬）

- 今日成本率（月結成本結構卡已有月級「支出佔收入比」；日級成本率單日波動大、參考價值低）

- **E2E（Playwright）**：
  - `smoke.spec.ts`：移除「首頁」tab 斷言與 Hero 檢查，改驗帳目頁小計三張卡渲染。

- `tsc --noEmit`、`npm test`、`npm run build` 全綠。

## 錯誤處理

- 未知 `categoryId`（同步帶入的陌生類別）不計入平台費與佔比，
  與既有 `dayIncome/dayExpense` 只計已知類別的防污染原則一致。
- 金額皆為正數（`Transaction` 既有不變式），平台費以 `Math.max(0, …)` 防禦。

## 不做的事（YAGNI）

- 不做首頁「瘦身保留」的折衷（方案 B 已否決）。
- 不動月曆格淨額定義（維持毛額，Phase 7 已評估過的刻意差異——
  月曆掃視用毛額、單日小計新卡才是扣分潤實收）。
- 不新增設定開關（不提供「顯示/隱藏首頁」選項）。
